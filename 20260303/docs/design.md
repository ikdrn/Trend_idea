# 20260303 設計メモ — trend-mixer-cli

## 1. 目的
トレンド情報を読むだけで終わらせず、
「次に何を作るか（行動）」に変換する。

## 2. 入力
- トレンド名
- 解決したい課題
- 実装の方向性

CLIでは `--trend "名前|課題|方向性"` 形式で受け取る。

## 3. 出力
- システム名
- キャッチコピー
- 解く課題
- 実装方針
- 1〜3時間で作る最小ステップ

## 4. 最小機能
1. 引数を受け取る（`argparse`）
2. 入力を構造化する（`dataclass`）
3. テンプレートを使って提案文を作る
4. コンソールに見やすく表示する

## 5. 今後の拡張
- Markdown保存機能（`--save`）
- 優先度スコアリング
- 「技術スタック別テンプレート」の追加
