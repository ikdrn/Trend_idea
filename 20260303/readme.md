## 【本日のシステムアイデア】

### アイデアの着想プロセス
- トレンドA: JS/TS ランタイムの継続的な人気
- トレンドB: セキュリティ情報の優先順位づけの難しさ
- 掛け合わせ: トレンドを入力すると、すぐ行動できる「小さな実装計画」に変換するCLI。

### 本日のシステム: `trend-mixer-cli` — トレンドを“次の行動”に変える

#### システム概要
- 入力: 2つ以上のトレンド（名前 / 課題 / 実装方針）
- 処理: トレンドを組み合わせ、1〜3時間で作れるシステム案を作成
- 出力: システム名、課題、実装方針、最小ステップ

#### 技術選定の理由（Step 3.5 反映）
- 今回は **JavaScript（Node.js）** を採用。
- 理由: JS/TS 系のトレンド継続に合わせつつ、この環境で `node` がすぐ使えるため。
- 初学者向けに外部依存をゼロにし、1コマンドで実行できるようにした。

#### 動作イメージ
```bash
node src/main.js --demo
```

出力例（要約）
```text
Trend Mixer CLI v2.0 (JavaScript)
==========================================
システム名: 速報ウォッチャー
キャッチコピー: ニュースを読むだけで、次にやることが決まる
...
```

## 実行方法（3ステップ以内）

```bash
# 1) デモ実行（おすすめ）
node src/main.js --demo

# 2) 自分のトレンドを指定
node src/main.js \
  --trend "JS/TSツール|学習対象が多すぎる|用途別に比較する" \
  --trend "脆弱性対応|優先順位が難しい|危険度で並べる"

# 3) パターン変更（任意）
node src/main.js --demo --pattern 2
```

## 学習ポイント（初学者向け）
- 引数処理: `process.argv` でCLI入力を扱える
- 配列処理: `map` / `join` で文字列整形を簡潔に書ける
- まずは `--demo` で動作確認してから、`--trend` を自分の内容に変えると理解しやすい
