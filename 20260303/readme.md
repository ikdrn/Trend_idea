## 【本日のシステムアイデア】

### アイデアの着想プロセス
- トレンドA: AIエージェント活用の拡大
- トレンドB: セキュリティ情報の増加と対応優先順位の難しさ
- 掛け合わせ: 「トレンド情報を入れると、優先度付きの小さな実行計画を返す」CLIを作る。

### 本日のシステム: `trend-mixer-cli` — トレンドを“次の行動”に変える

#### システム概要
- 入力: 2〜3個のトレンド（名前 / 課題 / 実装方針）
- 処理: トレンドを組み合わせて、1〜3時間で作れるシステム案を生成
- 出力: システム名、目的、最小実装ステップ、期待される成果

#### 技術選定の理由
- Python（標準ライブラリのみ）で実装し、初学者でもすぐ実行できるようにした。
- 依存ライブラリ不要なので、`python3 src/main.py --demo` ですぐ試せる。

#### 動作イメージ
```bash
python3 src/main.py --demo
```

出力例（要約）
```text
Trend Mixer v1.0
========================================
システム名: 速報ウォッチャー
キャッチコピー: ニュースを読むだけで、次にやることが決まる
...
```

## 実行方法

```bash
# デモデータで実行（おすすめ）
python3 src/main.py --demo

# 自分でトレンドを指定して実行
python3 src/main.py \
  --trend "AIエージェント運用|導入判断が難しい|用途別に分類する" \
  --trend "脆弱性対応|情報が多く優先順位が難しい|危険度と期限を整理する"
```

## 学習ポイント（初学者向け）
- `argparse`: コマンドライン引数を扱う標準機能
- `dataclass`: データ構造を短く書ける便利機能
- まずは `--demo` で動かし、その後 `--trend` の値を変えて挙動を確認すると理解しやすい
