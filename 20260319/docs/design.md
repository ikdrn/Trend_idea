## 2. 要件定義

### 2.1 背景

2026年3月19日、Hacker News にて「Show HN: Interactive 3D three-body problem simulator in browser」がトップ議論に浮上した。同日 GitHub Trending では `newton-physics/newton`（GPU物理シミュレーション）が 345★/day を獲得。物理シミュレーションをブラウザ上でインタラクティブに動かす需要が高まっており、教育・可視化ツールとしての有効性が改めて注目されている。

### 2.2 目的

- N体重力問題をブラウザ上でリアルタイムに可視化する
- 誰でも直感的に天体を追加・操作できるインタラクティブな体験を提供する
- 外部ライブラリ・APIキー不要で、HTMLファイル1つで即起動できる

### 2.3 対象範囲

- フロントエンドのみ（サーバーなし）
- N体重力シミュレーション（Nは動的、上限なし）
- 5種のプリセット軌道
- パラメーターのリアルタイム調整

### 2.4 用語定義

| 用語 | 説明 |
|------|------|
| N体問題 | N個の天体が相互に重力を及ぼし合う系の運動を求める問題。N≥3で解析解が存在しない（カオス性） |
| ソフトニング | 天体間距離が0に近づいたとき力が発散しないよう分母に定数を加える近似 |
| リープフロッグ積分 | 速度と位置を半ステップずらして交互に更新する、エネルギー保存に優れた数値積分法 |
| 8の字軌道 | Chenciner & Montgomery (2000) が発見した3体の周期的安定解 |
| トレイル | 各天体の過去位置を記録し残像として描画したもの |

---

### 2.5 利用者

| ユーザー種別 | 説明 |
|-------------|------|
| 一般ユーザー | 天体物理・カオスに興味を持つ非専門家。直感的操作で楽しむ |
| 学生・教育者 | 重力・軌道力学の学習教材として利用 |
| 開発者 | Canvas API・物理シミュレーション実装の参考として閲覧 |

---

### 2.6 業務概要

**業務フロー**

1. ユーザーがブラウザで `index.html` を開く
2. デフォルトで8の字軌道プリセットが起動
3. クリック/ドラッグで天体を追加・初速設定
4. スライダーでリアルタイムにパラメーター調整
5. プリセットボタンで異なる軌道パターンに切り替え

**業務課題**

- 三体問題は解析解がないため数値積分が必須
- 天体が近接したとき力が発散しシミュレーションが破綻する問題（ソフトニングで解決）
- 多天体時のパフォーマンス（O(N²)の力計算）

**性能**

- 20天体以下で 60FPS 維持（モダンブラウザ）
- requestAnimationFrame で VSyncに同期、不必要な再描画なし

**可用性**

- ブラウザのみで動作（Node.js・サーバー不要）
- オフライン動作可能

**セキュリティ**

- 外部通信なし。ローカルのみで完結
- eval・危険な DOM 操作なし

**運用・保守**

- 1ファイル構成（HTML/CSS/JS 統合）のため保守が容易

## 4. 詳細設計

### 4.1 モジュール一覧

| モジュールID | モジュール名 | 概要 |
|------------|------------|------|
| M-01 | Body クラス | 天体の状態（位置・速度・加速度・トレイル・質量・色） |
| M-02 | computeForces | 全ペア O(N²) 重力計算。ソフトニング適用 |
| M-03 | integrate | Velocity Verlet 積分。境界ラップ処理 |
| M-04 | draw | Canvas 描画（トレイル・グロー・ドラッグプレビュー） |
| M-05 | preset | 5種初期配置（8の字・太陽系・連星・カオス・銀河） |
| M-06 | イベント処理 | mouse/touch/keyboard イベントハンドリング |
| M-07 | メインループ | requestAnimationFrame + 可変 dt 管理 |

---

### 4.2 クラス設計

| クラス名 | 概要 |
|---------|------|
| Body | 天体1つを表すデータクラス |

**属性**

| 属性名 | 型 | 説明 |
|--------|-----|------|
| x, y | number | 位置（px） |
| vx, vy | number | 速度（px/s） |
| ax, ay | number | 加速度（フレーム毎にリセット） |
| mass | number | 質量（重力計算・半径に影響） |
| radius | number | 描画半径（= cbrt(mass) × 3） |
| color | string | 16進カラーコード |
| trail | Array<{x,y}> | 過去位置の配列（最大 trailLen 件） |

**メソッド**

| メソッド名 | 引数 | 戻り値 | 説明 |
|-----------|------|--------|------|
| constructor | x,y,vx,vy,mass,color | Body | 初期化 |

---

### 4.3 処理フロー

```
requestAnimationFrame(loop)
  │
  ├─ dt = min(elapsed, 50ms)
  │
  ├─ if (!paused)
  │    └─ for steps in ceil(simSpeed):
  │         computeForces()  // O(N²)
  │         integrate(subdt)
  │
  └─ draw()
       ├─ fillRect (残像ブラー)
       ├─ 各 body のトレイル描画（グラデーション）
       ├─ 各 body の描画（glow + core）
       └─ ドラッグプレビュー（点線 + 矢印）
```

### 4.7 例外処理

- dt > 50ms: `Math.min` でクランプ（タブ切替時のスパイク防止）
- ゼロ除算: ソフトニング係数でr²に定数加算し防止
- Canvas リサイズ: `resize()` ハンドラで `canvas.width/height` を再設定
